Component Building Guidelines

For every component we build we follow the above documentation sections within Figma

Composition

The component itself, as well as its location and use in our product system, are briefly described at the beginning of this section. The component's construction information is shown on a blueprint in the second section along with the corresponding labels.

Intent

This section lists the various variations of a component that are offered. When thinking about an alert component, a good example to grasp this is if the component conveys any intent or provides feedback to the users: It can represent a variety of purposes (positive, negative, neutral, etc.)

Attributes

Properties and attributes display the features that a specific component gives to its users. They also aid in imposing restrictions on it, such as specifying a text layer's character limit or the necessary and optional fields. In order to maintain a single source of truth, this section was created and is filled with input from design system engineers.

Design Token

The token table lists all the tokens and components that are used inside a specific component, if any. This enables developers to gain a comprehensive insight from just one frame.

Components & Variants

The master component itself, along with all appropriate versions that follow all defined rules, is the last item in this section.
